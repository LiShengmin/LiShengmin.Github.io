    SDK：iOS 11.0+ Beta

    框架

    IdtentifyLookUp

    创建可以识别和过滤不需要的短信和彩信的应用程序扩展，并保留用户隐私。

    概览

    当收到来自未知发件人的SMS或MMS消息时，Messages应用程序可以询问您的应用程序扩展以确定消息是否需要过滤。您的应用程序扩展可以通过使用自己的内置数据和逻辑或通过延迟关联的服务器进行的分析来做出这一决定。

    要查看来自未知发件人的邮件是否需要过滤，“信息”应用将启动当前启用的邮件过滤器应用并进行查询，如图1所示。

    将查询传递给消息过滤器应用程序扩展以获取消息的状态

    “信息”应用将使用一个对象，将有关该消息的信息传递到您的消息过滤器应用程序扩展。如果您的应用程序扩展可以确定消息不需要，它会将其决定返回给对象中的消息。
    ILMessageFilterQueryRequest
    ILMessageFilterQueryResponse
    如果您的应用扩展程序本身无法做出这一决定，则会告知消息将有关该消息的信息发送到与您的应用相关联的服务器。您的服务器会检查邮件信息，并发送回邮件，该邮件将响应传递给您的应用程序扩展。应用程序扩展解析服务器的响应，并给出对象中消息的最终决定，如图2所示。
    ILMessageFilterQueryResponse
    图2
    将查询传递给应用程序的关联服务器以获取消息的状态

    注意

    出于隐私的原因，系统处理与您的关联服务器的所有通信; 您的邮件过滤器应用程序扩展名无法直接访问网络。
    另外，出于隐私的原因，您的应用扩展程序无法将数据写入与包含的应用程序共享的容器。
    主题
    第一步

    创建消息过滤器应用程序扩展
    创建一个可以识别不需要的邮件的应用程序扩展。
    应用程式扩充功能
    class ILMessageFilterExtensionContext
    消息过滤应用程序扩展的扩展上下文。
    class ILMessageFilterExtension
    消息过滤器应用程序扩展的主要类的抽象基类。
    查询
    class ILMessageFilterQueryRequest
    请求消息过滤应用程序扩展，以确定从未知发件人接收的消息的状态。
    protocol ILMessageFilterQueryHandling
    一组通过消息过滤应用程序扩展来处理查询请求的方法。
    回应
    class ILMessageFilterQueryResponse
    对消息过滤器查询请求的响应。
    class ILNetworkResponse
    响应HTTPS代理消息过滤器应用程序扩展执行的HTTPS网络请求。
    enum ILMessageFilterAction
    采取响应收到的消息的动作。
    错误
    struct ILMessageFilterError
    let ILMessageFilterErrorDomain: String
    enum ILMessageFilterError.Code
